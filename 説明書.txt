==========================================================
  マウスカーソルをタッチ用サブモニタに行かせないツール
==========================================================

Corsair Xeneon Edge などのタッチ対応サブモニタに、マウスカーソルが
入り込むのを防ぐツールです。
タッチ操作はこれまで通り使えます(タッチはブロックしません)。

インストールは不要です。フォルダを置いて、ファイルをダブルクリック
するだけで使えます。

配布元: https://github.com/yum-hub/xeneon-edge-mouse-block

----------------------------------------------------------

■ 最初に:zip でダウンロードした人へ(重要)

ネット経由で受け取った zip は、Windows が安全のために「ブロック」
マークを付けることがあります。そのまま展開すると動かないことが
あるので、展開する前に次をしてください。

 1. ダウンロードした zip ファイルを右クリックして「プロパティ」を開く

 2. 一番下に「セキュリティ: このファイルは他のコンピューターから
    取得したものです…」と表示されていたら、「許可する」
    (または「ブロックの解除」)にチェックを入れて「OK」を押す

    ※ この表示が無ければ、そのまま次へ進んで大丈夫です

 3. zip を右クリック →「すべて展開」で、好きな場所
    (例: ドキュメント)に展開する

展開したフォルダはどこに置いても構いませんが、後で移動しない場所に
置くのがおすすめです。

----------------------------------------------------------

■ 使い方

 ・ブロックを開始する … start.vbs をダブルクリック
 ・ブロックを止める   … stop.vbs をダブルクリック

 ・どちらも画面には何も表示されません(裏で静かに動きます)。
 ・start.vbs を2回押しても大丈夫です(二重には起動しません)。
 ・パソコンを再起動するとブロックは解除されます。毎回自動で
   有効にしたい場合は、下の「Windows 起動時に自動で開始する」を
   行ってください。

----------------------------------------------------------

■ 動いているか確認したいとき

マウスをタッチ画面の方向へ動かしてみてください。
カーソルが手前の画面の端で止まれば動作しています。
タッチ操作は普通にできます。

----------------------------------------------------------

■ どのモニタがブロックされるの?

自動で決まります。

 1. Xeneon Edge(解像度 2560x720、縦置きなら 720x2560)があれば
    自動で選ばれます。ほとんどの人は何もしなくて OK です。

 2. 見つからない場合は、モニタを選ぶ画面が出ます。
    一覧には次のように番号・解像度・位置が表示されます。

        モニタ 1: 2560x1080  位置(0,0) [メイン画面]
        モニタ 2: 2560x720   位置(0,1080)

    タッチ画面(カーソルを入れたくない方)を選んで OK を
    押してください。

    ・どれがどのモニタか分からないときは、Windows の
      「設定 → システム → ディスプレイ」を開いて「識別」ボタンを
      押してください。各モニタの画面に大きな番号が表示されるので、
      それと見比べて選べます。

    ・選んだ結果はフォルダ内の config.json に保存され、次回からは
      聞かれません。

    ・選び直したいときは config.json を削除してから start.vbs を
      実行してください。もう一度選ぶ画面が出ます。

----------------------------------------------------------

■ Windows 起動時に自動で開始する

 1. start.vbs を右クリックして「その他のオプションを確認」→
    「ショートカットの作成」を選ぶ(ショートカットができます)

 2. キーボードの Windows キー + R を押す

 3. 出てきた小さい窓に shell:startup と入力して Enter を押す
    (「スタートアップ」フォルダが開きます)

 4. 手順1で作ったショートカットを、その開いたフォルダに
    ドラッグして移動する

これで、次回から Windows にサインインすると自動でブロックが
始まります。
やめたいときは、スタートアップフォルダのショートカットを削除して
ください。

----------------------------------------------------------

■ アンインストール(やめたいとき)

 1. stop.vbs をダブルクリックして止める
 2. スタートアップ登録をした場合は、shell:startup フォルダの
    ショートカットを削除する
 3. このフォルダごと削除する

レジストリなどには何も書き込まないので、ファイルを消すだけで
完全に元通りになります。

----------------------------------------------------------

■ しくみ(かんたんに)

 ・Windows の「低レベルマウスフック」という仕組みで、マウスの移動を
   見張っています。
 ・カーソルが対象モニタの領域へ入ろうとしたら、その移動を
   キャンセルして手前の画面の端で止めます
   (上下左右・斜め、どの配置でも動きます)。
 ・タッチやペンの操作は、Windows が付ける専用の目印で見分けて、
   一切ブロックしません。

----------------------------------------------------------

■ うまく動かないとき・知っておいてほしいこと

 ・管理者権限で動いているアプリ(タスクマネージャーなど)が最前面の
   ときは、Windows の仕様上ブロックが効かないことがあります。

 ・モニタの配置や解像度を変えた場合、Xeneon Edge(2560x720)なら
   自動で追従します。それ以外のモニタを選んでいた場合は、
   config.json を削除して選び直してください。

 ・タッチ/ペンの見分けは Windows 標準の目印に頼っています。ごく一部の
   特殊なドライバやアプリが作る「偽のマウス操作」は、マウス扱いに
   なってブロックされることがあります。

 ・ゲームなど、マウスを独自に制御するアプリとは相性問題が出ることが
   あります。その場合は stop.vbs で一時的に止めてください。

 ・モニタが 1 台しかないパソコンでは、安全のため起動しません
   (お知らせのメッセージが出ます)。

 ・zip の「ブロックの解除」を忘れていると、start.vbs を押しても何も
   起きないことがあります。上の「最初に」の手順をやり直してください。

 ・何か問題が起きたときは、フォルダ内に error.log というファイルが
   できていないか見てください。あれば、その中身を配布元まで
   知らせてもらえると原因が分かりやすくなります。

----------------------------------------------------------

■ ファイル一覧

 ・BlockMouseToEdge.ps1 … 本体(常駐して見張るプログラム)
 ・start.vbs            … 開始用(ダブルクリック)
 ・stop.vbs             … 停止用(ダブルクリック)
 ・stop.ps1             … 停止処理の本体(stop.vbs から呼ばれます)
 ・説明書.txt           … このファイル
 ・config.json          … 選んだモニタの記録(自動で作られます)
 ・error.log            … 問題が起きたときの記録(自動で作られます)
