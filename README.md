# マウスカーソルをタッチ用サブモニタに行かせないツール

Corsair Xeneon Edge などのタッチ対応サブモニタに、
**マウスカーソルが入り込むのを防ぐ**ツールです。
**タッチ操作はこれまで通り使えます**(タッチはブロックしません)。

インストールは不要です。フォルダを置いてダブルクリックするだけで使えます。

## ダウンロード

**[Releases ページ](https://github.com/yum-hub/xeneon-edge-mouse-block/releases/latest)** を開いて、
`XeneonEdgeMouseBlock.zip` をクリックしてダウンロードしてください。
ダウンロードしたら、すぐ下の「最初に:zip でもらった人へ」の手順に進んでください。

## 最初に:zip でもらった人へ(重要)

ネット経由で受け取った zip は、Windows が安全のため「ブロック」マークを付けることがあります。
そのまま展開すると動かないことがあるので、**展開する前に**次をしてください。

1. ダウンロードした zip ファイルを**右クリック**して「**プロパティ**」を開く
2. 一番下に「**セキュリティ: このファイルは他のコンピューターから取得したものです…**」と
   表示されていたら、「**許可する**」(または「ブロックの解除」)に**チェック**を入れて「OK」
   ※ この表示が無ければ、そのまま次へ進んで大丈夫です
3. zip を右クリック →「**すべて展開**」で好きな場所(例: ドキュメント)に展開する

展開したフォルダはどこに置いても構いませんが、**後で移動しない場所**に置くのがおすすめです。

## 使い方

| やりたいこと | 操作 |
|---|---|
| ブロックを開始する | `start.vbs` をダブルクリック |
| ブロックを止める | `stop.vbs` をダブルクリック |

- どちらも画面には何も表示されません(裏で静かに動きます)。
- `start.vbs` を2回押しても大丈夫です(二重には起動しません)。
- パソコンを再起動するとブロックは解除されます。毎回自動で有効にしたい場合は、下の「スタートアップ登録」を行ってください。

## どのモニタがブロックされるの?

自動で決まります。

1. **Xeneon Edge(解像度 2560x720、縦置きなら 720x2560)があれば自動で選ばれます。**
   ほとんどの人は何もしなくて OK です。
2. 見つからない場合は、**モニタを選ぶ画面**が出ます。
   一覧には「モニタ 1: 2560x1080 位置(0,0) [メイン画面]」のように
   番号・解像度・位置が表示されるので、**タッチ画面(カーソルを入れたくない方)**を
   選んで OK を押してください。
   - どれがどれか分からないときは、Windows の「設定 → システム → ディスプレイ」で
     「識別」ボタンを押すと、各モニタに番号が表示されるので参考にしてください。
   - 選んだ結果はフォルダ内の `config.json` に保存され、次回からは聞かれません。
   - 選び直したいときは `config.json` を削除してから `start.vbs` を実行してください。

## 動いているか確認したいとき

マウスをタッチ画面の方向へ動かしてみてください。
カーソルが手前の画面の端で止まれば動作しています。タッチ操作は普通にできます。

## Windows 起動時に自動で開始する(スタートアップ登録)

1. `start.vbs` を右クリックして「**その他のオプションを確認**」→「**ショートカットの作成**」を選ぶ
   (ショートカットファイルができます)
2. キーボードの **Windows キー + R** を押す
3. 出てきた小さい窓に `shell:startup` と入力して **Enter**
   (「スタートアップ」フォルダが開きます)
4. 手順1で作ったショートカットを、その開いたフォルダに**ドラッグして移動**する

これで、次回から Windows にサインインすると自動でブロックが始まります。

## アンインストール(やめたいとき)

1. `stop.vbs` をダブルクリックして止める
2. スタートアップ登録をした場合は、`shell:startup` フォルダのショートカットを削除する
3. このフォルダごと削除する

レジストリなどには何も書き込まないので、**ファイルを消すだけ**で完全に元通りです。

## しくみ(かんたんに)

- Windows の「低レベルマウスフック」という仕組みで、マウスの移動を監視しています。
- カーソルが対象モニタの領域へ入ろうとしたら、その移動をキャンセルして
  手前の画面の端で止めます(上下左右・斜め、どの配置でも動きます)。
- タッチやペンの操作は、Windows が付ける特別な目印
  (`dwExtraInfo` の署名 `0xFF5157xx`)で見分けて、**一切ブロックしません**。

## 注意・既知の制限

- モニタの配置や解像度を変えた場合、Xeneon Edge(2560x720)なら自動で追従します。
  それ以外のモニタを選んでいた場合は、`config.json` を削除して選び直してください。
- タッチ/ペンの見分けは Windows 標準の署名(`0xFF515700`)に頼っています。
  ごく一部の特殊なドライバやアプリが作る「偽のマウス操作」は、
  マウス扱いになってブロックされる可能性があります。
- 管理者権限で動いているアプリ(タスクマネージャーなど)が最前面のときは、
  権限の関係でブロックが効かないことがあります(Windows の仕様です)。
- ゲームなど、マウスを独自に制御するアプリとは相性問題が出ることがあります。
  その場合は `stop.vbs` で一時的に止めてください。
- モニタが 1 台しかないパソコンでは、安全のため起動しません(メッセージが出ます)。
- 何か問題が起きたときは、フォルダ内に `error.log` ができていないか見てください
  (問い合わせの際に添えてもらえると原因が分かりやすくなります)。
